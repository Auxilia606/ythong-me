# Documentation

This directory is organized by document purpose.

## Structure

- `architecture/`: frontend architecture rules, layer boundaries, and implementation conventions.
- `api/`: backend integration requirements and API-facing contracts.
- `feature-context/`: feature-level context summaries and reusable documentation templates.
- `deployment/`: deployment and operations documentation.
- `plans/`: product and implementation planning documents that are not permanent architecture rules.

## Current Planning Docs
